# 领域边界

本文档定义 Honeycomb 中最重要的四个边界：任务、角色、技能、进化。

## 1. 任务边界

任务是一次具体工作的运行容器，负责：

- 当前目标
- 当前输入与上下文
- 当前拓扑
- 当前资源限制
- 当前运行状态

任务不负责：

- 定义长期能力
- 决定全局默认实现
- 决定长期晋升与淘汰

一句话：任务回答“现在要完成什么”。

## 2. 角色边界

角色只属于执行面。

当前最小角色：

- `queen`：蜂后，负责解释任务、派发工作、汇总状态
- `worker`：工蜂，负责执行单个派发单元

角色不等于能力，也不等于技能定义。

## 3. 技能边界

技能是长期能力契约，负责定义：

- 能力名称
- 输入输出协议
- 使用约束
- 评估指标
- 可演化范围

技能不等于任务，不等于某次运行实例，也不等于角色。

一句话：技能回答“这个蜂巢长期能做什么”。

## 4. 进化边界

进化是长期治理系统，负责：

- 汇总运行结果
- 生成评分
- 维护谱系
- 晋升或淘汰实现体
- 沉淀最佳实践

进化不负责：

- 执行当前任务
- 直接调度当前工蜂
- 在任务运行时随意修改长期定义

一句话：进化回答“未来应该保留什么”。

## 5. 统一理解

- 任务：短期运行容器
- 角色：当前运行职责
- 技能：长期能力契约
- 进化：长期优化与治理

这四者必须始终分开，否则后续协议、存储和权限模型都会混乱。
