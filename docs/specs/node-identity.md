# 节点身份

节点是 bee 运行时（`honeycomb-bee`）中的最小运行实例。它可以是蜂后，也可以是工蜂。

## 1. 节点身份要解决什么问题

节点身份需要回答：

- 这个实例是谁
- 它属于哪个任务
- 它属于哪个租户和命名空间
- 它的上级是谁
- 它运行的是什么二进制和协议版本

## 2. 建议字段

建议每个节点至少包含：

- `node_id`
- `role`
- `task_id`
- `tenant_id`
- `namespace`
- `owner_user_id`
- `owner_team_id`
- `hive_id`
- `capability`
- `parent_node_id`
- `queen_node_id`
- `binary_id`
- `binary_version`
- `protocol_version`
- `status`

## 3. 关键原则

- 节点身份属于 bee 运行时，不属于进化面
- 节点身份是运行时实例身份，不是长期技能身份
- 工蜂必须继承任务、租户和命名空间边界
- 运行时血缘与进化谱系不是同一件事

## 4. 运行时血缘

运行时血缘描述：

- 哪个蜂后派生了哪个工蜂
- 哪个工蜂属于哪个任务
- 工蜂之间是否存在父子关系

这条血缘只用于运行时协作与恢复。

## 5. 与进化谱系的区别

进化谱系描述的是：

- 哪个实现体来自哪个父实现体
- 哪条实现分支被晋升或淘汰
- 哪个最佳实践来自哪些历史结果

它不等于运行时节点之间的派生关系。
