# 多租户模型

## 1. 目标

多租户模型定义 Honeycomb 如何在一个部署中隔离多个租户。

## 2. 核心原则

租户是最高层隔离边界。

默认规则：

- 先按租户隔离
- 再按命名空间限制可见性
- 最后再判断权限动作

## 3. 与多用户模型的关系

- 租户在最外层
- 用户和团队属于某个租户
- `global` 在多租户中默认表示租户内全局，而不是跨租户全局

## 4. 需要携带的字段

建议以下对象都带上：

- `tenant_id`
- `owner_user_id`
- `owner_team_id`
- `namespace`
- `visibility`

## 5. 存储原则

建议目录层次先按租户，再按命名空间。

## 6. Bee 运行时与租户规则

- 蜂后只能在同一租户内派生工蜂
- 工蜂只能连接同一租户、同一任务的蜂后
- 运行态不得跨租户读取

## 7. 进化面规则

- 评分与晋升默认租户内闭环
- 用户到团队、团队到租户内全局的晋升必须显式进行
- 跨租户共享不属于第一版
