# 模板与最佳实践联动

## 1. 目标

本规范定义任务模板、拓扑模板与最佳实践档案之间的关系，避免后续把“任务组织经验”和“技能使用经验”混成一层。

## 2. 两类经验的区别

### 2.1 模板

模板回答：

- 这类任务一般怎么组织
- 默认使用什么拓扑
- 一般会用到哪些技能组合

### 2.2 最佳实践

最佳实践回答：

- 某个技能在某类上下文下更适合怎么用
- 推荐哪个实现体
- 推荐哪些参数或协作顺序

## 3. 联动原则

建议关系如下：

- 模板可以引用最佳实践
- 最佳实践可以影响模板的默认值
- 但模板不等于最佳实践
- 最佳实践也不应直接生成任务模板

## 4. 推荐任务启动流程

建议任务启动时：

1. 先选择或匹配任务模板
2. 展开拓扑模板
3. 对模板中涉及的技能逐个匹配最佳实践
4. 生成本次任务的实际执行计划

## 5. 模板引用最佳实践的方式

建议模板中保留结构位，例如：

- 某个节点引用“优先匹配最佳实践”
- 某个节点固定使用某个实现体
- 某个节点允许覆盖默认实现体

这样模板和最佳实践可以组合，但不会绑死。

## 6. 最佳实践反向影响模板

当某类最佳实践长期稳定后，可以反向推动模板更新，例如：

- 某个技能在该类任务中默认应使用另一实现体
- 某个拓扑节点应新增一个前置步骤
- 某个常见约束应成为模板默认值

但这种更新应经过模板层的显式版本更新，而不是隐式自动漂移。

## 7. 作用域与发布

模板和最佳实践都应遵守：

- 租户边界
- 命名空间边界
- 权限与发布流程

例如：

- 用户级最佳实践不应直接改团队级模板
- 团队级模板更新不应自动影响全局模板

## 8. 第一版建议

第一版建议做到：

- 模板支持引用最佳实践
- 启动任务时可做一次模板加最佳实践匹配
- 模板更新保持显式版本化

第一版不必承诺：

- 自动模板改写
- 模板与最佳实践双向实时联动
- 大规模模板推荐引擎

## 9. 总结

模板与最佳实践应被视为两层不同但可协作的长期经验资源：

- 模板沉淀任务组织方式
- 最佳实践沉淀技能使用方式

它们联动，但不混同。
