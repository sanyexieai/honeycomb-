# 任务与调度器规范

## 1. 目标

本规范定义任务如何被接收、排队、推进和完成。

## 2. 任务层级并行

建议区分三层并行：

### 2.1 任务级并行

多个任务同时运行。

### 2.2 蜂巢级并行

同一任务中多个派发单元或多个蜂巢会话并行。

### 2.3 实现体竞争并行

同一能力的多个实现体为了评估而并行试跑。

这第三层默认不应成为业务主路径。

## 3. 调度器职责

调度器负责：

- 接收任务
- 检查配额与资源限制
- 维护任务状态
- 推进 ready 派发单元
- 记录中间状态
- 在失败时触发恢复策略

## 4. 任务状态建议

建议最少状态：

- `queued`
- `running`
- `completed`
- `failed`
- `cancelled`
- `interrupted`

## 5. 调度原则

- 先检查租户与命名空间边界
- 再检查资源与配额
- 再推进拓扑 ready 节点
- 优先复用空闲工蜂
- 所有调度决策都要可追踪

## 6. 第一版建议

第一版先做到：

- 单机内任务运行
- 任务级并行
- 基于拓扑的 ready 推进
- 配额拒绝或允许
- 失败时有限重试

更复杂的公平调度和预测调度后续再做。
