# 技能导入导出流程

## 1. 目标

本规范定义技能包在系统中的实际流转流程。

重点不是包格式本身，而是从“本地技能”到“可发布技能”，再到“可回滚技能”的操作路径。

## 2. 导出流程建议

建议流程：

1. 从注册表定位技能
2. 确认导出版本与作用域
3. 收集技能定义和相关资源
4. 生成 `manifest.json`
5. 计算摘要
6. 生成包目录或压缩包
7. 写入导出审计记录

## 3. 导入流程建议

建议流程：

1. 读取包元数据
2. 校验目标租户与命名空间
3. 校验版本与兼容性
4. 处理命名和版本冲突
5. 写入目标技能目录
6. 更新技能注册表
7. 写入导入审计记录

## 4. 发布流程建议

从较小范围发布到较大范围时，建议视为“导出 + 导入”的受控变体。

例如：

- 用户空间导出
- 团队空间导入
- 注册表更新
- 审计与晋升记录写入

## 5. 回滚流程建议

回滚建议流程：

1. 定位目标历史版本或历史包
2. 校验当前作用域是否允许回滚
3. 切换注册表默认版本
4. 保留当前版本历史，不直接抹除
5. 写入回滚审计记录

## 6. 关键原则

- 导入导出和发布都必须是显式动作
- 不允许无记录的静默覆盖
- 包流转不应绕过注册表
- 回滚优先切换注册表状态，而不是直接修改运行时目录

## 7. 第一版建议

第一版优先支持：

- 本地导出
- 本地导入
- 用户到团队、团队到全局的显式发布
- 版本回指和审计记录

更复杂的远程同步、签名校验、仓库镜像后续再做。
