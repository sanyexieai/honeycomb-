# 执行面与进化面

## 1. 为什么要拆开

Honeycomb 需要同时处理两类问题：

- 当前任务怎么执行
- 长期能力如何优化

这两类问题的时间尺度、状态模型、权限边界都不同，因此应拆成两个平面。

## 2. 执行面

执行面负责：

- 创建任务
- 启动蜂后和工蜂
- 执行任务拓扑
- 记录运行事件
- 产出结果与评估输入

执行面的关键特征：

- 高频
- 短期
- 任务级隔离
- 关注当前完成度

## 3. 进化面

进化面负责：

- 汇总执行结果
- 计算评分
- 维护实现谱系
- 决定晋升、淘汰、沉淀最佳实践

进化面的关键特征：

- 低频或批量
- 长期
- 跨任务汇总
- 关注长期默认选择

## 4. 二者之间的数据关系

执行面输出：

- 任务结果
- 运行事件
- 追踪记录
- 审计记录
- 评估输入

进化面消费这些数据，并输出：

- 评分报告
- 晋升记录
- 淘汰记录
- 谱系更新
- 最佳实践更新

## 5. 边界原则

- 执行面不直接改长期技能定义
- 进化面不直接干预正在运行的任务
- 两者通过文件、记录或显式接口连接
- 长期状态写入由进化面负责

## 6. 当前建议

第一版优先做稳：

- 执行面最小闭环
- 执行到进化的数据接口
- 进化面的离线或批量处理

不要一开始就做强耦合实时闭环。
