# 蜂后与工蜂模型

## 1. 基本定义

Honeycomb 执行面由同一二进制的不同实例组成。

- 蜂后：任务内的局部协调者
- 工蜂：被蜂后派生并接受派发的执行者

## 2. 蜂后职责

蜂后负责：

- 启动任务
- 解释任务拓扑
- 派生工蜂
- 下发派发单元
- 汇总结果与状态
- 结束任务并回收工蜂

## 3. 工蜂职责

工蜂负责：

- 向蜂后注册
- 接收单个派发单元
- 报告进度
- 返回结果
- 接受关闭命令

工蜂不负责理解整个任务图，也不负责长期治理。

## 4. 组织原则

- 单任务内允许蜂后主导
- 系统级不设唯一全局中心
- 工蜂默认只服务一个任务
- 工蜂默认只连接一个蜂后

## 5. 当前建议

第一版先支持：

- 同机通信
- 蜂后派生工蜂
- 工蜂直连蜂后
- 任务结束后工蜂退出

更复杂的自治协作以后再扩展。
