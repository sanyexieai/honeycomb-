# 多用户模型

## 1. 目标

多用户模型定义 Honeycomb 如何表示不同用户、团队、命名空间以及它们之间的共享边界。

## 2. 最小对象

建议至少有：

- `User`
- `Team`
- `Namespace`
- `Owner`
- `Visibility`

## 3. 命名空间层级

建议三层：

- `global`
- `team/<team_id>`
- `user/<user_id>`

## 4. 归属建议

### 4.1 任务

任务归属于某个用户，可选归属于某个团队。

### 4.2 蜂后与工蜂

蜂后与工蜂继承任务的用户、团队和命名空间边界。

### 4.3 技能

技能应支持按命名空间发布和隔离。

### 4.4 进化结果

进化结果默认先在用户或团队范围内沉淀，不建议直接全局公开。

## 5. 原则

- 默认先私有，再共享
- 用户级实验不直接污染全局
- 团队共享应显式晋升
- 任何共享都应保留来源和审计记录
