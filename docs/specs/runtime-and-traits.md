# 运行时与 Trait 草案

## 1. 目标

本规范用于整理 Rust 侧的核心数据结构与 Trait 边界，作为后续实现参考。

## 2. 推荐核心结构

建议围绕以下对象建模：

- 技能定义
- 实现体定义
- 基因定义
- 任务规格
- 任务运行时
- 任务蜂巢会话
- 输入输出对象
- 评分与晋升对象

## 3. 推荐核心 Trait

建议至少包含：

- `Hive`
- `Executor`
- `Scheduler`
- `HiveRepository`
- `PracticeRepository`
- `Orchestrator`
- `EvolutionManager`

## 4. 设计原则

- Trait 边界应围绕系统边界而不是围绕临时实现
- 运行态对象与长期态对象分离
- bee 运行时 Trait 与进化面 Trait 不要混在一起
- 存储抽象要先于具体实现形式

## 5. 当前建议

第一版代码实现时，优先确保：

- 任务边界清晰
- 节点身份清晰
- 执行与进化接口分离
- 文件系统实现与接口定义分离
