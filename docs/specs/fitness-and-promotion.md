# 评分与晋升

## 1. 目标

本规范定义执行结果如何进入评估，再如何进入晋升或淘汰流程。

## 2. 三层流程

### 2.1 执行层

产出任务结果、指标、事件和上下文。

### 2.2 评估层

把执行结果转换成可比较的评分报告。

### 2.3 晋升层

根据评分决定：

- 晋升实现体
- 晋升最佳实践
- 淘汰表现差的分支

## 3. 核心对象

建议至少包含：

- `FitnessReport`
- `PromotionRecord`
- `DeprecationRecord`

## 4. 晋升原则

- 先候选，后晋升
- 先在较小范围验证，再晋升到更大范围
- 晋升必须保留证据链
- 晋升必须有审计记录

## 5. 当前建议

第一版优先支持用户级和团队级范围内的显式晋升，不急着做复杂自动治理。
