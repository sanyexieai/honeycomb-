# 权限与可见性

## 1. 目标

本规范定义谁能看到什么，以及谁能做什么动作。

## 2. 两个不同概念

### 2.1 可见性

可见性回答：谁可以看到某个对象。

### 2.2 权限

权限回答：谁可以对某个对象执行某个动作。

## 3. 评估顺序

建议顺序：

1. 租户边界
2. 命名空间边界
3. 可见性判断
4. 权限动作判断

## 4. 最小可见性等级

- `private`
- `team`
- `global`

## 5. 最小动作集合

建议至少支持：

- `read`
- `execute`
- `write_runtime`
- `write_definition`
- `promote`
- `deprecate`
- `spawn_worker`
- `inspect`

## 6. 对象分类

建议至少覆盖：

- 任务
- 技能
- 进化结果
- 配额配置
- 审计记录

## 7. 关键原则

- 工蜂是执行代理，不是治理主体
- 权限判断不能跳过租户和命名空间边界
- 长期定义写入权限必须严格高于运行时写权限
