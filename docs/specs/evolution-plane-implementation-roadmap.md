# 进化面最小实现路线图

## 1. 目标

本规范把进化面从设计概念收敛成一条最小可实现路线。

目标不是立刻做出完整自治进化系统，而是先建立一个能消费执行结果、生成评分、形成治理决策、并更新长期状态的最小闭环。

## 2. 实现原则

建议遵守：

- 先离线或批量处理，再考虑实时化
- 先评分和记录，再做自动晋升
- 先显式治理，再做复杂自动化
- 先支持回溯，再追求智能优化

## 3. 第一阶段目标

第一阶段只要求做到：

- 能读取 bee 运行时输出的评估输入
- 能生成 `FitnessReport`
- 能写入 `evolution/` 目录
- 能形成最小晋升或废弃建议
- 能通过注册层更新长期默认值

## 4. 推荐阶段划分

### 阶段 1：评估输入读取

建议实现：

- 读取 bee 运行时评估输入
- 建立进化面基础对象
- 写入 `evaluations/`

### 阶段 2：评分生成

建议实现：

- `FitnessReport`
- 指标汇总
- 简单评分公式
- 历史评分记录

### 阶段 3：治理决策

建议实现：

- `promote`
- `hold`
- `deprecate`
- `observe`

### 阶段 4：注册表联动

建议实现：

- 生成注册表更新记录
- 更新默认实现体
- 更新最佳实践正式状态
- 保留回滚历史

### 阶段 5：谱系与最佳实践沉淀

建议实现：

- 谱系记录
- 最佳实践档案写入
- 作用域内发布

## 5. 第一版可暂缓

- 全自动全局晋升
- 复杂审批工作流
- 跨租户进化共享
- 实时在线治理闭环

## 6. 总结

进化面的最小路线应先完成：

执行结果 -> 评估输入 -> 评分报告 -> 治理决策 -> 注册表更新

先把这条链打通，再逐步引入更复杂的自我优化与生态演化能力。
